File System Usage Guide

Static Assets (CSS, JS, Images, Documents)

Store static assets like CSS, JavaScript, images, and documents in the assets/ folder within your project directory. These files are served directly from your GitHub Pages site.

  • CSS files: assets/css/
  • JavaScript files: assets/js/
  • Image files: assets/images/
  • Video files: assets/videos/
  • Document files: assets/documents/

User-Uploaded Content (Avatars, Gig Images, Resumes, etc.)

Store user-uploaded content like avatars, gig images, resumes, and cover letters in cloud storage (Firebase Storage or Amazon S3). This ensures scalability and security.

  • Avatars: avatars/{userId}/{timestamp}_{fileName}
  • Gig Images: gigs/{userId}/{timestamp}_{fileName}
  • Resumes: resumes/{userId}/{timestamp}_{fileName}
  • Cover Letters: cover_letters/{userId}/{timestamp}_{fileName}
